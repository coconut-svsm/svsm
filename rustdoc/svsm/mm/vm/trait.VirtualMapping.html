fn map(&self, offset: usize) -> Option<PhysAddr>

Request physical address to map for a given offset

§Arguments
  • offset - Offset into the virtual memory mapping
§Returns

Physical address to map for the given offset, if any. None is also a valid return value and does not indicate an error.

Source

fn pt_flags(&self, offset: usize) -> PTEntryFlags

Request the PTEntryFlags used for this virtual memory mapping.

§Arguments
  • ‘offset’ -> The offset in bytes into the VirtualMapping. The flags returned from this function relate to the page at the given offset
§Returns

A combination of:

  • PTEntryFlags::WRITABLE
  • PTEntryFlags::NX,
  • PTEntryFlags::ACCESSED
  • PTEntryFlags::DIRTY

fn handle_page_fault( &mut self, _vmr: &VMR, _offset: usize, _write: bool, ) -> Result<VMPageFaultResolution, SvsmError>

Handle a page fault that occurred on a virtual memory address within this mapping.

§Arguments

  • ‘vmr’ - Virtual memory range that contains the mapping. This VirtualMapping can use this to insert/remove regions as necessary to handle the page fault.

  • offset - Offset into the virtual mapping that was the subject of the page fault.

  • ‘write’ - true if the fault was due to a write to the memory location, or ‘false’ if the fault was due to a read.
